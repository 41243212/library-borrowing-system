# 新手安裝與執行教學

這份文件是給第一次使用 Git、Docker 的同學看的。照著做，就可以把專案從 Git 抓下來，開啟 Docker，並在瀏覽器看到圖書館借閱管理系統。

## 你會用到什麼

- Git：用來把專案從 GitHub 或 Git server 抓到你的電腦。
- Docker Desktop：用來啟動 PHP、MySQL、phpMyAdmin，不需要自己手動安裝 PHP 或 MySQL。
- 瀏覽器：用來打開系統畫面。
- 終端機：Mac 可用 Terminal；Windows 可用 PowerShell。

## 1. 安裝 Git

先確認電腦有沒有 Git。

```bash
git --version
```

如果有看到版本號，例如 `git version 2.x.x`，代表已經安裝。

如果沒有安裝：

- Windows：到 https://git-scm.com/download/win 下載並安裝。
- Mac：通常輸入 `git --version` 後，系統會提示安裝 Command Line Tools；照畫面安裝即可。

## 2. 從 Git 抓下專案
打開終端機，移動到你想放專案的地方，例如桌面：

```bash
cd Desktop
```

把專案抓下來：

```bash
git clone https://github.com/41243212/library-borrowing-system.git
```

進入專案資料夾：

```bash
cd library-borrowing-system
```

如果你看到 `Dockerfile`、`docker-compose.yml`、`src`、`database`，代表位置正確。

## 3. 安裝 Docker Desktop

請安裝 Docker Desktop，不要只裝 Docker CLI。

官方安裝文件：

- Mac：https://docs.docker.com/installation/mac/
- Windows：https://docs.docker.com/desktop/setup/install/windows-install/

安裝完成後，請打開 Docker Desktop。

注意：一定要看到 Docker Desktop 正在執行，接下來的指令才會成功。

可以用這個指令確認 Docker 是否能用：

```bash
docker --version
```

再確認 Docker Compose 是否能用：

```bash
docker compose version
```

如果兩個都有版本號，代表環境大致正常。

## 4. 用 Docker 啟動專案

確認你目前在專案資料夾內，也就是有 `docker-compose.yml` 的地方。

執行：

```bash
docker compose up --build
```

第一次執行會下載映像檔，會比較久，請耐心等。

如果看到類似下面的訊息，代表服務正在跑：

```text
csie_team14_app
csie_team14_mysql
csie_team14_phpmyadmin
```

如果你想讓它在背景執行，可以用：

```bash
docker compose up --build -d
```

## 5. 打開系統畫面

Docker 啟動完成後，打開瀏覽器。

網站主程式：

```text
http://localhost:8080
```

phpMyAdmin：

```text
http://localhost:8082
```

手機掃描借還書頁：

```text
http://localhost:8080/mobile_scan.php
```

## 6. 登入帳號

系統內建兩組測試帳號。

管理員：

```text

密碼：admin123
```

讀者：

```text

密碼：reader123
```

如果你想自己註冊帳號，可以在登入頁按「註冊」。

## 8. 停止專案

如果你的終端機還停在 Docker 執行畫面，可以按：

```text
Ctrl + C
```

如果是背景執行，請用：

```bash
docker compose down
```

## 9. 重設資料庫

如果你把資料玩亂了，想回到最初狀態，可以重設資料庫。

注意：這會刪除目前 Docker 裡的 MySQL 資料。

```bash
docker compose down -v
docker compose up --build
```

## 10. 常見問題

### 打不開 localhost:8080

先確認 Docker Desktop 有開。

再確認服務是否有啟動：

```bash
docker compose ps
```

如果沒看到服務，重新執行：

```bash
docker compose up --build
```

### 出現 port already in use

代表你的電腦上已經有別的程式使用同一個 port。

本專案預設使用：

- Web：`8080`
- phpMyAdmin：`8082`
- MySQL：`3307`

可以先關掉其他程式，或請同學/助教協助修改 `docker-compose.yml` 的 port。

### Windows 執行 Docker 失敗

請確認 Docker Desktop 已安裝並啟動。Windows 通常也需要 WSL 2，Docker Desktop 安裝流程會提示你。

如果不確定，請重新打開 Docker Desktop，等它顯示 Running 或 Engine running 後再執行指令。
